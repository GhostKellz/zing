# Security Policy

## Supported Scope

Security reports are welcome for the current `zing` CLI and the code shipped in this repository.

The project is early-stage software. Reports that demonstrate a real impact in the current implementation are the most useful.

## Reporting

Please report security issues privately to `ckelley@ghostkellz.sh`.

Include:

- affected version or commit
- reproduction steps
- expected impact
- any suggested fix or mitigation

## Disclosure Expectations

- Please do not open a public issue for an unpatched vulnerability.
- Acknowledgement and triage should happen after the report is reviewed.
- Public disclosure should wait until a fix or mitigation is available.
